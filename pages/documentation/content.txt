<h3>Are You Serious?</h3>
<br />
<strong>OK, then small FAQ:</strong><br /><br />

<ul>
<li><strong>How to make template?</strong><br />
It is Your will. Use <pre>/templates/</pre>folder. Every folder inside marks as templae.<br /><br />
</li>

<li><strong>What are pages if there is no DB?</strong><br />
PHP file of Page and TXT file of Properties.<br />
If You need anything else -- Use Your Favorite Framework!
<br />
<br />
</li>

<li><strong>How to use Page Properties?</strong>
<p>We are storing some values for dynamic content there.</p>
<p>For example, there is some code for menu items, or property relative content.</p>
</li>

<li><strong>How to make News?</strong>
<p>Use directory based structure.</p>
<pre> /pages/news/YYYY/MM/DD/Number </pre>
<p>Make links and landings.</p>
<br />
<br />
</li>

<li><strong>What with other content? I need e-mail, rss, geoIP, maps, web-services...</strong>
<p>As for webservices, any page could be marked as <strong>Server Side Code</strong><br />
 at Pages Manager => Settings tab.</p><br />

<p>As for the rest -- <strong>Write Your own dude, write Your own...</strong><br />
We are using <pre>/components/</pre>
for storing some regular code (menu, json decoder).<br />
All that in there availiable as <pre>$success['components']</pre> inside of the template.</p><br />
<br />

</li>
