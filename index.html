Concept

The main objective was to create the most simple cms for web sites, simple web sites.

Actually, this stuff must be not even like   CMS.

Goal of productivity is first of all we were thinking of.

Imagine!


For about 90% of websites we don't even need any DB!


All these sites made for only one mission â€“ to tell about something or someone.

Yes, they have a lot of dynamic content, like: news, photos, product line data and so on... Dynamic... yepp...

But, please, ask Youself, how much of this content is entierly necessity to be in such dynamic way?

Please.
We are well remembered about personal pages or organization promo sites with almost static news pages.

So, do we really need News management in DB?

Or do we really need Photo Album with odd storage and extremely hard management?

Or do we really need goods promo with diversed settings but which is not as well as complicated?

Then, may be we need to get deeper?

Current small FAQ


OR documentation:

  • How to make template?
    It is Your will. Use 
    /templates/
    folder. Every folder inside marks as templae.

  • What are pages if there is no DB?
    PHP file of Page and TXT file of Properties.
    If You need anything else -- Use Your Favorite Framework!

  • How to use Page Properties?

    We are storing some values for dynamic content there.

    For example, there is some code for menu items, or property relative content.

  • How to make News?

    Use directory based structure.

     /pages/news/YYYY/MM/DD/Number

    Make links and landings.



  • What with other content? I need e-mail, rss, geoIP, maps, web-services...

    As for webservices, any page could be marked as Server Side Code
    at Pages Manager => Settings tab.


    As for the rest -- Write Your own dude, write Your own...
    We are using 

    /components/
    for storing some regular code (menu, json decoder).
    All that in there availiable as 
    $success['components']
    inside of the template.
