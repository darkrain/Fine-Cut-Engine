First of all – what does it mean «Fine Cut»?

We got rid of everything we've found unnecessary to develop simple website.

John Lennon tald: «All You Need is Love».
We say: «all we need is UI».

Then, for complete understanding Fine Cut it is not enough to read this documentation.


However, we suppose that there will be a lot of people who don't ever need complete understanding.


But, if you wish, you need to investigate only one file: «_engine/index.php»

Sombody will tell: "Are You Serious?"


OK, here is small FAQ:

  • What are pages if there is no DB?
    PHP file of Page and TXT files of its Properties.
    If You need anything else -- Use Your Favorite Framework!

  • What with other content? I need e-mail, rss, geoIP, maps, web-services...

    As for webservices, any page could be marked as «non cacheable»
    at Pages Manager => Settings tab.


    As for the rest – write your own code.
    Please, lookup for examples page, if not yet.

  • You told about perfomance, did you?

    I could tell that on my nettop with AMD C-60 CPU inside, exactly this page generation process passthrough in ~0.05 sec. And while that generation, page stores in static. Therefore, when i will request page again, i will get it after ~0.007 sec. I use standard HDD 5400 rpm, and didn't any software optimisatoins.
