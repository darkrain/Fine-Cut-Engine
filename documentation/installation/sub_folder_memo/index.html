If you'll work with Fine Cut inside of «sub folder» of «document root» folder you shoud keep in mid that you are working inside.

By defauld everything in Fine Cut is independent, components and default templates made irrelevant to any filesystem path.

Please, learn how we did this before creatin own.

And, here are some advices:

  • Do not use direct links (with http://), when linking between one Fine Cut site pages. Use relative ones. Relative links are paths that you see at Pages Manager tab Tree (in the left side):

    There is also a button, with hint «Copy this Leaf Path» to be sure you are using quite well link.
    And, yes, you could open such as many «_adm» pages as it necessary.

  • Exactly the same as for [Pages Manager][Page Editor], correct path means for every own resource inside of the Fine Cut based website: each URL must be done without path to directory where Fine Cut is installed. Just look how we did it.

  • If you wish complete explanation: we use simple pre-processor based on regular expressions to rename every link or source attribute inside page content or page header additional data. That pre-processors are two components named as «content_parser» and «head_additional». These components are extremely simple, they just replaces everything with «http="/» or «src="/» by adding sub folder path before link. So, when your link or resource is inside of Fine Cut, just be sure it starts from fine cut installation dir. And, yes, if you will get to create own template – use these pre-processors for compatibility.
